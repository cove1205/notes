# 开源许可证选择指南总结

核心观点：

> **一个开源许可证可以保护贡献者和使用者**，**商业公司和有经验的开发者不会使用没有许可证的项目**。

## 选择指南

根据你的情况选择：

| 场景         | 推荐许可证                                                        | 特点                                                               |
| ------------ | ----------------------------------------------------------------- | ------------------------------------------------------------------ |
| 需要社区协作 | [社区偏好许可证](https://choosealicense.com/community/)           | 使用项目社区已经在用的许可证                                       |
| 想要简单宽松 | **MIT License**                                                   | 短小精悍，允许他人几乎任何用途，**包括创建分发闭源版本**           |
| 关心改进分享 | **GNU GPLv3**                                                     | 允许几乎任何用途，**但分发闭源版本必须公开源码**，保证改进也能共享 |
| 非软件项目   | [专门许可证](https://choosealicense.com/non-software/)            | 文案、图片、数据等非软件项目也有对应许可证                         |
| 需要更多选择 | [完整许可证列表](https://choosealicense.com/licenses/)            | 更多许可证可供选择                                                 |
| 不想选择     | [看看没有许可证会怎样](https://choosealicense.com/no-permission/) | 默认保留所有版权，他人无法贡献、修改、分发你的项目                 |

## 常见项目案例

| 许可证 | 使用案例                     |
| ------ | ---------------------------- |
| MIT    | Babel、.NET、Rails           |
| GPLv3  | Ansible、Bash、uBlock Origin |

## 许可证列表

### 1. GNU AGPLv3

这款最强Copyleft许可证的许可权限，条件是：如果你使用本许可软件进行更大规模的项目开发，修改后的完整源代码必须以相同许可证公开。

**权限**

- ✓ 商业使用
- ✓ 分发
- ✓ 修改
- ✓ 专利使用
- ✓ 私人使用

**条件**

- ⓘ 必须公开源代码
- ⓘ 保留许可证和版权声明
- ⓘ 网络使用也算分发
- ⓘ 相同许可证
- ⓘ 声明变更

**限制**

- ✕ 不承担责任
- ✕ 不提供保修

[查看完整 GNU Affero General Public License v3.0 »](https://choosealicense.com/licenses/agpl-3.0/)

### 3. GNU LGPLv3

这款Copyleft许可证的许可权限，条件是：修改后的完整源代码必须以相同许可证公开，但是通过接口使用本许可代码的大型项目可以不同许可证分发，不需要公开源代码。

**权限**

- ✓ 商业使用
- ✓ 分发
- ✓ 修改
- ✓ 专利使用
- ✓ 私人使用

**条件**

- ⓘ 必须公开源代码
- ⓘ 保留许可证和版权声明

- ⓘ 声明变更

**限制**

- ✕ 不承担责任
- ✕ 不提供保修

[查看完整 GNU Lesser General Public License v3.0 »](https://choosealicense.com/licenses/lgpl-3.0/)

---

### 4. Mozilla Public License 2.0

这款弱Copyleft许可证的许可权限，条件是：修改后的许可文件必须以相同许可证（某些情况下可以使用GNU许可证）公开源代码。版权和许可证声明必须保留。贡献者授予专利权。但是，使用本许可代码的大型项目可以不同许可证分发，不需要公开新增文件的源代码。

**权限**

- ✓ 商业使用
- ✓ 分发
- ✓ 修改
- ✓ 专利使用
- ✓ 私人使用

**条件**

- ⓘ 必须公开源代码
- ⓘ 保留许可证和版权声明
- ⓘ 相同许可证（文件级）

**限制**

- ✕ 不承担责任
- ✕ 商标使用
- ✕ 不提供保修

[查看完整 Mozilla Public License 2.0 »](https://choosealicense.com/licenses/mpl-2.0/)

这是一个宽松许可证，主要条件是保留版权和许可证声明。贡献者授予专利权。许可的作品、修改、大型项目可以以不同条款分发，不需要公开源代码。

**权限**

- ✓ 商业使用
- ✓ 分发
- ✓ 修改
- ✓ 专利使用
- ✓ 私人使用

**条件**

- ⓘ 保留许可证和版权声明

**限制**

- ✕ 不提供保修

[查看完整 Apache License 2.0 »](https://choosealicense.com/licenses/apache-2.0/)

- ✓ 分发
- ✓ 修改

### 8. The Unlicense

没有任何条件，直接放入公共领域。未授权的作品、修改、大型项目可以任意分发，不需要源代码。

**权限**

- ✓ 商业使用
- ✓ 分发
- ✓ 修改
- ✓ 私人使用

**限制**

- ✕ 不承担责任
- ✕ 不提供保修

[查看完整 The Unlicense »](https://choosealicense.com/licenses/unlicense/)

## 总结

以上许可证涵盖了从高度保护到无条件的整个范围，对于大多数新项目来说，其中一个应该就够用了。许多其他开源许可证也存在，包括上述许可证的旧版本和相近替代品。
